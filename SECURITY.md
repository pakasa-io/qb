# Security Policy

## Reporting A Vulnerability

If you believe you have found a security issue, do not open a public issue with
exploit details.

Use GitHub's private vulnerability reporting for the repository if it is
enabled. If private reporting is not available yet, contact the repository
owner through their published GitHub or organization contact channel and share
only the minimum detail needed to start a secure follow-up.

## What To Include

Please include:

- the affected package or feature
- the supported versions you tested
- clear reproduction steps
- impact assessment
- any suggested remediation if you already have one

## Response Expectations

Maintainers will triage reports, confirm impact, and prepare a fix or
mitigation before public disclosure when possible.

For non-sensitive hardening questions, regular issues and pull requests are
still appropriate.
